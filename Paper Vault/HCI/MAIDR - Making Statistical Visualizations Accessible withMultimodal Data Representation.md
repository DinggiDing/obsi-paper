# 3 CO-DESIGN PROCESS AND CONSIDERATIONS

- **DC1**: 시스템은 처음에 널리 사용되는 시각화를 지원해야 한다.
    - 막대 그래프, 히트맵, 상자 그림, 산점도 등의 시각화는 널리 사용되며 중요한 통계적 개념을 나타내는 대표적인 시각화로 선택되었습니다.
- **DC2**: 점자 디스플레이에 적합한 확장 가능한 촉각 표현이 제공되어야 한다.
    - MAIDR 시스템은 단일 줄 점자 디스플레이(RBD)와 호환되도록 설계되었으며, 이 디스플레이의 낮은 해상도 한계를 해결하기 위해 8점 점자를 사용하여 시각화의 패턴과 모양을 촉각적으로 표현했습니다.
- **DC3**: 각 데이터 포인트는 표준 4방향 화살표 키로 탐색할 수 있어야 한다.
    - 사용자는 화살표 키를 사용해 데이터를 탐색하며, 데이터 간의 관계를 분석할 수 있습니다.
- **DC4**: 혼합된 모드의 결합은 사용자가 설정할 수 있어야 하고, 모드 간 커서 위치가 연동되어야 한다.
    - 시스템은 다양한 모드(점자, 텍스트, 소리)를 조합하여 사용자가 선호하는 방식으로 데이터를 탐색할 수 있으며, 이 과정에서 통일된 커서 위치가 유지됩니다.
- **DC5**: 각 모드에서 제공되는 정보는 사용자에 의해 조정될 수 있어야 한다.
    - 사용자는 점자 디스플레이의 열 크기, 소리의 음역대, 텍스트의 상세도 등을 조정하여 자신에게 맞는 방식으로 데이터를 접근할 수 있습니다.
- **DC6**: 겹쳐진 요소를 순차적으로 탐색할 수 있는 계층 탐색이 가능해야 한다.
    - 복잡한 데이터 시각화에서 여러 계층의 요소가 겹칠 경우, 시각장애인이 이를 효율적으로 탐색할 수 있도록 계층을 순차적으로 탐색하는 기능이 필요합니다.

# 4 THE MAIDR SYSTEM

**사용자 상호작용 및 탐색**: MAIDR 시스템은 사용자가 시각화 이미지를 탐색할 수 있는 웹 기반 인터페이스를 제공합니다.

- **입력 파일**: 시스템은 플롯 이미지(그래프)와 해당 이미지의 메타데이터 스키마(JSON 형식)를 필요로 합니다. 이 메타데이터에는 그래프의 유형, 제목, 축의 라벨 및 각 데이터 포인트의 정보가 포함됩니다.
- **웹 브라우저 지원**: MAIDR는 **파이어폭스, 구글 크롬, 마이크로소프트 엣지, 사파리** 등 대부분의 웹 브라우저에서 실행할 수 있습니다. 사용자는 생성된 HTML 파일을 브라우저에서 열어 시각화된 데이터를 상호작용할 수 있습니다.
- **탐색 방법**: 사용자는 시각화된 그래프에서 **탭(Tab) 키**를 눌러 워크스페이스를 선택한 후, **화살표 키**를 사용해 데이터 포인트를 탐색할 수 있습니다.
    - X축의 데이터를 탐색할 때는 좌우 화살표를 사용하고, Y축은 상하 화살표로 탐색합니다.
    - 상자 그림(box plot) 및 산점도(scatter plot)와 같은 **다중 데이터 포인트**를 가진 그래프의 경우, 여러 값을 그룹으로 묶어 탐색할 수 있도록 설계되었습니다.
    - **PageUp/PageDown** 키는 계층화된 요소(예: 산점도의 예측선) 간의 레이어 전환을 돕습니다.

**도움말 메뉴**:

- 시스템은 **H 키**로 호출할 수 있는 도움말 메뉴를 제공합니다. 이 메뉴는 탐색 방법에 대한 설명뿐만 아니라 사용자 정의 옵션을 제공합니다. 예를 들어, **점자 디스플레이의 크기**를 조정하거나 **소리화 모드의 음역대**를 설정할 수 있습니다.
- 기본적으로 모든 모드(점자, 텍스트, 소리, 검토)는 꺼져 있으며, 사용자가 원하는 모드를 수동으로 활성화해야 합니다.
